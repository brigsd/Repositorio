# Instruções para IAs que trabalham neste projeto

## Antes de começar QUALQUER trabalho

1. **Leia `DEVLOG.md`** na raiz do projeto — é a memória compartilhada entre todas as ferramentas
2. **Leia `docs/README.md`** para entender o índice da documentação
3. **Se a tarefa envolve uma área específica**, leia o documento correspondente em `docs/`

## Após concluir QUALQUER alteração

1. **Adicione uma entrada no topo** da seção "Entradas" do `DEVLOG.md`
2. **Use o formato padronizado** (ver template no DEVLOG)
3. **Identifique-se**: qual ferramenta (Antigravity, Claude Code, Manual) e de qual local (trabalho, casa)
4. **Liste TODOS os arquivos alterados** com descrição curta do que mudou
5. **Registre o estado atual** e o próximo passo sugerido

## Convenções do projeto

- **Documentação viva** mora em `docs/` — decisões importantes são registradas lá
- **Decisões de design** vão em `docs/11-decisoes-design.md` com número sequencial (D-01, D-02...)
- **Mudanças no roadmap** vão em `docs/09-roadmap.md` no changelog
- **Commits descritivos**: mensagem clara do que mudou e por quê
- **Banco de dados**: Neon (cloud) — sem Postgres local. Schema em `src/db/schema.ts`
- **Comandos de instalação** (npm install, etc.) devem ser passados ao usuário para rodar manualmente (conta admin necessária)
- **TailwindCSS** é o framework de estilo — respeite isso, não use CSS vanilla
- **Travessão (—)** não deve aparecer em textos que o aluno lê (ver D-01)
- **Termos gramaticais formais** não devem aparecer em explicações pedagógicas (ver D-02)

## Estrutura rápida

```
docs/           → Documentação pedagógica e técnica (12 arquivos)
src/app/        → Rotas Next.js (App Router)
src/db/         → Schema Drizzle + seed
src/lib/        → Lógica: IA, prompts, modelo-aluno, spaced repetition, currículo
DEVLOG.md       → Histórico de alterações entre sessões de IA
```
